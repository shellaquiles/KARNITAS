# Skill: SDD Implement

**Fase 5–6** · Inputs: `spec.md`, `plan.md`, `tasks.md`

## Implement

1. Cargar `always_load` + spec/plan/tasks; respetar alcance del implementer en spec
2. Ejecutar solo tareas asignadas a `[implementer]` (u otro rol si la tarea lo indica)
3. Tests por criterio EARS; validar antes de siguiente tarea
4. PR con `refs specs/NNN-nombre/spec.md`

## Iterate

1. Cambio de comportamiento → actualizar `spec.md` primero
2. Re-ejecutar Clarify/Plan/Tasks según el delta
3. Regenerar o ajustar tests

Gate: revisión humana. No merge autónomo.
