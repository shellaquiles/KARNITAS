# Skill: SDD Tasks

**Fase 4** · Input: `plan.md`, `spec.md` · Output: `tasks.md`

1. Derivar tareas desde plan, EARS y **paralelización** acordada en spec
2. Tareas atómicas: **1–4h**, un PR razonable
3. Asignar agente responsable por tarea: `[specifier]` `[planner]` `[implementer]` (default: implementer)
4. Marcar `[P]` solo si la spec lo autoriza y no hay estado compartido
5. Por tarea: ID, objetivo, agente, inputs, outputs, done (EARS), estimación
6. Última tarea: validar tests EARS

Gate: revisión de alcance, agentes y orden.
